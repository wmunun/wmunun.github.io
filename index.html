SQL注入

定义：用户输入的语句被当作代码执行

mysql 在 5.0 以上的版本加入了 information_schema 的系统自带库，其中保存了关于 mysql 服务器所维护的所有其他数据库的信息，如数据库名、数据库的表、表栏的数据类型与访问权限等。其中 tables 保存了所有表名，columns 保存了所有字段信息。

information_schema (库)

schemate (表) 存储该用户创建的所有数据库的库名

schema_name (字段) 记录数据库的库名

tables 存储该用户创建的所有数据库的库名和表名

table_schema 记录数据库库名

table_name 记录表名

columns 存储该用户创建的所有数据库的库名、表名、字段名

table_schema 记录数据库库名

table_name 记录表名

column_name 记录字段名

如果mysql版本低于5.0则需要爆破表名（吃字典）

表名错误则会报错

爆破出表名后接着爆破字段名

3、宽字节注入

GBK 双字节编码 ASCII 单字节编码

涉及到的编码格式均与数据库有关，与前后端无关

但是前段为GBK那么后端为GBK的可能性非常大

思路：当计算机以GBK编码进行处理时，将输入的内容与反斜杠拼接为一个汉字，使单引号逃逸出去

URL编码为 十六进制 反斜杠的十六进制编码为 %5c 此时拼接 %df 即可拼接为汉子 “运”

select 语句中 无法使用双引号 即 无法用 where 指定条件

1、套娃

2、mysql支持十六进制输入，即用十六进制替代字符串进行输入

使用sqlmap跑宽字节注入 需要手动加 %df

POST传参进行宽字节注入

%df 是URL编码后的结果，而POST传参中一般不会进行URL编码

使用sqlmap跑POST宽字节注入则需抓包，用星号指定注入点

方法一

使用BURP修改hex中的十六进制的值
